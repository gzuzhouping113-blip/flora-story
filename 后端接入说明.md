# 花历后端接入说明

## 当前技术栈

- Next.js App Router：页面入口和 API Routes。
- Prisma：数据库访问层。
- Neon Postgres：当前线上数据库，连接串通过 `DATABASE_URL` 配置。
- 图片存储：本地开发默认保存到 `public/uploads`，线上可切换到 Cloudinary 或 Cloudflare R2。
- 火山方舟 / 豆包：真实模式下用于视觉识别、文案生成和图生图。

## 运行方式

```bash
npm install --ignore-scripts
npx prisma generate
npx prisma db push
npm run dev
```

访问：

```text
http://127.0.0.1:3000/flora_story.html
```

## 环境变量

本地联调可以继续使用 mock 模式，不消耗模型额度，但数据库已经使用 Neon/Postgres。

```env
DATABASE_URL="postgresql://USER:PASSWORD@HOST/neondb?sslmode=require&channel_binding=require"
AI_PROVIDER="mock"
EMAIL_PROVIDER="mock"
STORAGE_PROVIDER="local"
PUBLIC_APP_URL="http://127.0.0.1:3000"
```

接入真实豆包时，将 `.env.local` 配成：

```env
DATABASE_URL="postgresql://USER:PASSWORD@HOST/neondb?sslmode=require&channel_binding=require"
AI_PROVIDER="ark"
ARK_API_KEY="你的火山方舟 API Key"
ARK_BASE_URL="https://ark.cn-beijing.volces.com/api/v3"
ARK_IMAGE_MODEL="doubao-seedream-5-0-260128"
ARK_VISION_MODEL="doubao-seed-2-0-lite-260428"
PUBLIC_APP_URL="你的 Vercel 域名"
STORAGE_PROVIDER="cloudinary"
CLOUDINARY_CLOUD_NAME="你的 Cloudinary Cloud Name"
CLOUDINARY_API_KEY="你的 Cloudinary API Key"
CLOUDINARY_API_SECRET="你的 Cloudinary API Secret"
EMAIL_PROVIDER="resend"
RESEND_API_KEY="你的 Resend API Key"
MAIL_FROM="Flora Story <你的已验证发件邮箱>"
```

如果之后有银行卡或 PayPal，也可以切换到 Cloudflare R2：

```env
STORAGE_PROVIDER="r2"
R2_ACCOUNT_ID="你的 Cloudflare Account ID"
R2_ACCESS_KEY_ID="你的 R2 Access Key ID"
R2_SECRET_ACCESS_KEY="你的 R2 Secret Access Key"
R2_BUCKET_NAME="flora-story"
R2_PUBLIC_BASE_URL="https://你的-r2-public-domain"
```

配置后先运行：

```bash
npm run check:ark
```

图片存储配置后运行：

```bash
npm run check:storage
```

邮件配置后运行：

```bash
npm run check:email
```

如果要真实发送一封测试邮件：

```bash
npm run check:email:live -- your-email@example.com
```

如果只是确认文字/视觉模型接口是否连通，可以运行：

```bash
npm run check:ark:live
```

`check:ark:live` 会调用一次真实模型，可能产生少量费用；它不会测试图生图，避免误触发高成本图片生成。

注意：真实图生图时，豆包需要能访问原图 URL。本地 `127.0.0.1` 地址无法被模型服务访问；线上使用 Cloudinary 或 R2 后，上传接口会返回公开可访问的图片地址。

## 图片存储

当前存储层支持三种模式：

- `STORAGE_PROVIDER="local"`：本地开发使用，图片写入 `public/uploads/original` 和 `public/uploads/generated`。
- `STORAGE_PROVIDER="cloudinary"`：无银行卡时推荐，图片写入 Cloudinary，数据库保存公开图片 URL。
- `STORAGE_PROVIDER="r2"`：有付款方式后可选，图片写入 Cloudflare R2，数据库保存公开图片 URL。

R2 的公开地址必须使用 `https://pub-xxxx.r2.dev` 或自定义域名，不能使用 `https://<account-id>.r2.cloudflarestorage.com` 这个 S3 API endpoint。
删除花束记录时，后端会同步清理没有被其他记录引用的原图和生成图。

## 邮箱验证码

当前邮件层支持两种模式：

- `EMAIL_PROVIDER="mock"`：本地开发使用，验证码打印在服务端日志里。
- `EMAIL_PROVIDER="resend"`：线上使用，通过 Resend 发送真实验证码邮件。

上线前需要在 Resend 完成：

- 添加并验证发件域名。
- 按 Resend 提示配置 DNS 记录，等待域名状态变成 Verified。
- 创建 API Key，推荐使用 Sending access，并限制到你的发送域名。
- 设置 `MAIL_FROM` 为已验证域名下的邮箱，例如 `Flora Story <noreply@yourdomain.com>`。

生产环境不能使用 `onboarding@resend.dev` 作为正式发件人。为了减少接口滥用，同一个邮箱 60 秒内不能重复发送验证码，1 小时最多发送 5 次。

## API 列表

| API | 方法 | 用途 |
| --- | --- | --- |
| `/api/uploads` | POST | 上传原图到当前图片存储 |
| `/api/ai/generate-record` | POST | 识别花材、生成标题/锐评、按风格图生图 |
| `/api/records` | GET | 获取花历记录 |
| `/api/records` | POST | 保存花束记录 |
| `/api/records/[id]` | DELETE | 删除自己的花束记录 |
| `/api/flora-book` | GET | 聚合花房图鉴数据 |
| `/api/auth/request-code` | POST | 发送邮箱验证码 |
| `/api/auth/verify-code` | POST | 验证邮箱验证码并登录 |

## Prompt 文件

```text
prompts/
  image/
    magnet.zh.md
    watercolor.zh.md
    polaroid.zh.md
  vision/
    analyze-bouquet.zh.md
```

后端会根据 `style` 读取对应图生图 prompt。识别与文案生成使用 `vision/analyze-bouquet.zh.md`，并通过 JSON Schema 校验输出。

## 测试

```bash
npm run test:backend
npm run build
```

`test:backend` 会启动一个临时 Next 服务，测试邮箱登录、上传、AI mock 生成、保存记录、查询列表、花房聚合和删除记录。
