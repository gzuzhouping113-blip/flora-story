# 花历 MVP 交互与 UI 优化说明

## 产品结构

花历分为三个核心模块：

- 花历：回看全部花束记录，以藤蔓时间线承载故事。
- 上传花束：上传照片、选择状态与日期、填写故事、选择生成风格，并触发 AI 生成。
- 花房：展示累计记录、偏好统计、纸短情长记录和花朵图鉴。

## 花历

筛选分为两组：

- 状态筛选：全部记录 / 收到 / 送出。
- 时间筛选：默认展示最新年份，例如 2026。

交互规则：

- 默认展示全部记录 + 最新年份。
- 状态筛选和时间筛选可以组合。
- 不显示“全部”年份按钮。
- 点击当前选中的年份，例如 2026，会取消年份筛选；再次点击则重新选择该年份。
- 取消 2025 固定入口，后续年份应由真实数据动态生成。
- 筛选无结果时展示：这一段时间还没有花开。
- 花束卡片沿藤蔓左右交错排列，点击后打开花朵详情页。
- 卡片点击提供轻微缩放回弹反馈。
- 花束卡片底部罗列最多 3 个主要花朵名称，不在卡片内展示花语。

详情页展示：

- 花束图片。
- 标题。
- 日期与收到/送出状态。
- AI 锐评。
- 最多 5 种主要花朵和花语。
- 用户写下的故事。

## 上传花束

入口为底部居中的加号按钮。

用户步骤：

1. 上传鲜花照片。
2. 选择收到花 / 送出花，默认收到花。
3. 通过日期拖动组件选择日期，默认今天。
4. 写下花束背后的故事，最多 300 字，可选。
5. 选择风格：立体冰箱贴 / 治愈水彩 / 复古拍立得 / 保留原图。
6. 点击开始生成魔法。
7. 查看 AI 生成结果并保存至花历。

日期组件规则：

- 默认值为今天。
- 向左拖动日期变早。
- 向右拖动日期接近今天。
- 日期区不显示“今天”标签，右侧使用“选择日期”入口。
- 点击“选择日期”或日期卡片本身，打开系统原生日期选择器。
- 不允许选择未来日期。
- 拖到未来时回弹，并提示：花还没有开到那一天。

AI 生成结果需要包含：

- 主要花朵识别，最多 5 种。
- 每种花约 20 字花语。
- 4-10 字标题。
- 20-35 字锐评。
- 风格化图片。

原图风格：

- 默认选中立体冰箱贴。
- “保留原图”展示在风格列表最后。
- 选择“保留原图”时不调用 AI。
- 点击继续后直接进入结果确认页，图片使用用户上传的原始照片。
- 原图记录的花材信息显示为“原图：不进行 AI 识别，保留真实照片”。

## 花房

花房用于总结和导航。

- 点击累计记录，跳回花历，并重置为全部记录 + 全部年份。
- 点击纸短情长，打开故事最长的花束详情。
- 点击深夜的思念，跳回花历并定位到对应花束卡片。
- 顶部数字大花束与花历记录图片保持一一对应；每张图使用同一花束图片，点击后跳回花历并定位到对应花束卡片。
- 图鉴按花朵出现次数排序。
- 图鉴支持分页圆点切换。
- 图鉴列表 UI 与花朵卡片风格统一，展示花朵名、花语和次数。

## 已优化的 UI 点

- 花历筛选从单排按钮改为两组清晰胶囊控件。
- 花朵名称和花语统一为轻量信息条，应用在花束卡片和花房图鉴中。
- 上传向导加入照片预览、真实禁用态、日期拖动、字数统计和风格选中态。
- 上传与详情 Bottom Sheet 限制为移动端宽度，桌面预览不再被拉满。
- 花房从静态统计升级为可点击导航和分页图鉴。
- 保存生成结果后会插入花历顶部，并提示：这束花已经长进花历里了。
